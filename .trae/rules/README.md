# 项目规则索引

本目录包含 Turborepo-Rsbuild 项目的所有规则和指南文档。

## 规则文档列表

### 1. [项目规则](./project_rules.md)
定义了项目的核心开发规则和最佳实践，包括：
- 开发规则（依赖管理、开发环境、构建规则等）
- 文档规则（存放位置、命名规范、分类规范等）
- 代码规范（代码风格、TypeScript 规范、组件开发规范等）
- 项目结构规则（应用结构、技术栈选择、模块联邦等）
- 特殊规则（Vue/React 应用规则、API 服务规则、提交规则等）
- 违规处理

### 2. [命令参考](./command_reference.md)
提供了项目中所有常用命令的快速参考，包括：
- 基础命令（依赖管理、开发与构建、代码检查等）
- 应用特定命令（Vue/React 面板、桌面应用、API 服务等）
- 包特定命令（UI 组件库、Hooks 库、Utils 库等）
- 环境配置
- 特殊命令（项目优化、清理模块等）
- 服务地址
- 常见问题解决
- 开发提示

## 如何使用这些规则

### 新团队成员
1. 首先阅读 [项目规则](./project_rules.md) 了解整体开发规范
2. 保存 [命令参考](./command_reference.md) 作为日常开发的手册
3. 根据所负责的应用/包，重点阅读相关部分的规则

### 日常开发
1. 开发前检查环境变量设置是否符合规则
2. 使用命令参考中的正确命令进行开发、构建和测试
3. 提交前确保代码符合所有代码规范

### 代码审查
1. 对照项目规则检查代码是否符合规范
2. 特别关注文档规则、代码规范和提交规则
3. 确保所有变更都有相应的文档更新（如需要）
