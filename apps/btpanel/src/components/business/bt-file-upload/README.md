# BtFileUpload 文件上传组件

支持大文件分片上传和断点续传的文件管理组件。

## 功能特性
- 拖拽上传支持
- 分片上传机制
- 实时进度显示
- 速度计算与预估
- 错误重试机制
- 虚拟滚动列表

## 基本用法

```vue
<template>
  <bt-file-upload :comp-data="{ path: '/www/wwwroot' }" />
</template>
```

## 配置参数
| 参数名         | 类型     | 默认值       | 说明                |
|-------------|---------|-------------|-------------------|
| compData    | object   | {}          | 配置数据对象，包含 path 路径 {path: string} |
| chunkSize   | number  | 104857600   | 分片大小(默认100MB)    |
| maxRetries  | number  | 3           | 最大重试次数           |
| concurrency | number  | 5           | 并发上传数            |

## 核心功能
| 功能                 | 说明                          |
|---------------------|-----------------------------|
| 多选上传             | 支持文件/文件夹批量上传          |
| 进度可视化           | 文件级/全局进度条显示             |
| 智能续传             | 网络中断后自动恢复上传            |
| 速度优化             | 动态计算上传速度与剩余时间          |
| 内存管理             | 虚拟滚动技术处理大文件列表          |

## 状态管理
| 状态值       | 说明                | 界面表现          |
|-------------|-------------------|-----------------|
| paused      | 等待上传            | 显示上传按钮        |
| uploading   | 上传中              | 显示进度与速度       |
| success     | 上传成功            | 显示统计信息        |
| error       | 上传失败            | 显示错误提示        |

## 最佳实践
1. **路径配置**：通过 compData.path 设置上传目录
2. **类型限制**：使用 accept 属性限制上传类型
3. **大文件处理**：超过 100MB 自动分片上传
4. **错误处理**：自动重试失败文件（最多3次）
5. **性能调优**：根据网络调整并发数
6. **断点续传**：刷新页面后自动继续上传

## 关联接口
- 上传接口：`/files?action=upload`
- 文件存在检查：`/files?action=check_exists`
- 目录创建：`/files?action=create_dir`

> 需配合文件管理模块使用
