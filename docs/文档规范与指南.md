# 文档规范与指南

## 📁 文档分类规范

根据 CLAUDE.md 要求，所有项目文档必须按照以下分类组织：

### 1. 开发文档 (`docs/开发文档/`)
- **内容**：开发流程、规范、设计方案、架构文档
- **命名规范**：使用中文，按重要性添加序号前缀
- **示例**：`01-认证系统设计.md`、`02-数据库设计.md`

### 2. API文档 (`docs/API文档/`)
- **内容**：API接口说明、参数文档、集成指南
- **命名规范**：使用中文，说明具体API用途
- **示例**：`01-用户认证API.md`、`02-文件管理API.md`

### 3. 构建文档 (`docs/构建文档/`)
- **内容**：构建流程、部署指南、CI/CD配置
- **命名规范**：使用中文，聚焦具体流程
- **示例**：`01-Docker部署指南.md`、`02-CI配置说明.md`

### 4. 任务文档 (`docs/任务文档/`)
- **内容**：开发任务、需求文档、优化清单
- **命名规范**：使用中文，明确任务目标
- **示例**：`01-功能需求文档.md`、`02-优化任务清单.md`

## 🏷️ 文档命名规范

### ✅ 推荐做法
- 使用中文命名，清晰表达文档内容
- 按重要性添加序号前缀（01-, 02-, 03-）
- 名称简洁明了，避免过长
- 使用有意义的描述性标题

### ❌ 避免做法
- 英文名称或随意命名
- 过于复杂或模糊的标题
- 缺少分类前缀或序号

## 🔄 文档维护规范

### 文档创建
1. 确定文档类型，选择对应的分类目录
2. 使用中文命名文档，遵循编号规范
3. 参考已有文档的格式和风格
4. 保持文档的时效性和准确性

### 文档更新
1. 定期检查文档的时效性
2. 及时更新过时的信息
3. 在项目结构发生变化时同步更新相关文档
4. 保持文档的准确性和一致性

### 导航维护
1. 更新相关README.md中的导航链接
2. 确保文档间的交叉引用有效
3. 维护文档索引和目录结构

## 🎯 文档质量标准

### 内容要求
- **准确性**：信息准确，反映最新状态
- **完整性**：覆盖所有重要方面
- **可读性**：清晰易懂，逻辑清晰
- **实用性**：提供可操作的指导

### 格式要求
- 使用Markdown格式
- 遵循统一的编码风格
- 包含适当的标题层级
- 提供代码示例和实际场景

## 📚 文档类型详解

### 开发文档
重点描述：
- 系统架构设计
- 开发规范和最佳实践
- 代码组织方式
- 核心功能实现方案

### API文档
重点描述：
- 接口定义和参数说明
- 请求响应示例
- 错误处理和状态码
- 集成指南和使用示例

### 构建文档
重点描述：
- 构建流程和脚本
- 部署环境和配置
- CI/CD管道配置
- 环境变量设置

### 任务文档
重点描述：
- 功能需求和规格说明
- 开发任务分解
- 优化建议和改进计划
- 问题和解决方案记录
